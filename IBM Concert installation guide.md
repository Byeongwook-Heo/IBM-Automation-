# IBM Concert 설치 가이드 (VM/컨테이너 공통 개요 + Workflow 포함)

> 문서 목적: IBM Concert를 신규 환경에 도입할 때 필요한 준비사항, 설치 절차, 그리고 운영팀 기준의 Workflow(승인/검증/운영 전환)를 한 번에 정리한 실무형 가이드입니다.

---

## 1) 설치 아키텍처 개요

- **IBM Concert Platform**
  - Concert 애플리케이션 서비스
  - 데이터 저장소(배포 토폴로지에 맞는 관리형 DB/스토리지)
  - 인증 연계(SSO/LDAP/OIDC 등)
- **연동 대상 시스템**
  - CI/CD, ITSM, Observability, 자산/구성관리 시스템
  - API 기반 양방향 연계를 위한 네트워크/권한 설계
- **접속 계층**
  - 사용자 브라우저 → HTTPS 엔드포인트(FQDN)
  - 운영 자동화 배치/워크플로 엔진 → Concert API

---

## 2) 사전 준비사항

### 2.1 필수 체크리스트

- [ ] 설치 유형 확정(POC / Production)
- [ ] 라이선스 및 설치 자산 접근 권한 확보
- [ ] FQDN, DNS, TLS 인증서 준비
- [ ] 사용자 인증 연동 정책(로컬/SSO) 확정
- [ ] 방화벽/보안그룹 정책 반영
- [ ] 백업 및 복구 시나리오 수립
- [ ] 운영 계정(Role/Permission) 모델 정의

### 2.2 권장 환경 기준

- Production은 고가용성(HA) 가능한 토폴로지 권장
- 스토리지/DB는 리텐션 및 연계 데이터량 기준으로 용량 산정
- 시간 동기화(NTP) 및 로그 표준 포맷(감사 추적)을 사전 적용

### 2.3 보안 기준

- 전 구간 TLS 적용(내부 연동 포함)
- 비밀정보(API Key, Token, Password)는 Vault/Secret Manager 사용
- 최소 권한 원칙(RBAC): 운영자/개발자/감사 계정 분리

---

## 3) 설치 절차

> 실제 명령, 설치 패키지, 이미지 태그, 포트, 시스템 요구사항은 계약된 IBM Concert 버전의 공식 가이드를 최우선으로 확인하세요.

### 3.1 인프라 준비

1. 노드/VM(또는 Kubernetes 클러스터) 준비
2. 네트워크 라우팅/방화벽 규칙 적용
3. DNS 및 인증서 배포
4. 필수 런타임/의존성 패키지 점검

### 3.2 Concert 배포

1. 설치용 서비스 계정 생성
2. 설치 번들(또는 Helm/Operator 등 배포 방식) 적용
3. DB/스토리지 연결 정보 주입
4. 초기 관리자 계정 및 기본 조직/프로젝트 생성
5. 헬스체크 API 및 UI 접속 검증

### 3.3 초기 설정

- SSO/IdP 연동 구성
- 알림 채널(메일/메신저/웹훅) 설정
- 감사 로그 및 보존기간 정책 설정
- 기본 대시보드/리포트 템플릿 생성

---

## 4) 운영 Workflow (권장)

아래 Workflow는 운영 환경 도입 시 변경관리와 품질 검증을 포함한 표준 흐름입니다.

### 단계 1: Request & Scope 정의

- 신규 도입 요청 접수
- 대상 서비스/팀/연동 시스템 범위 확정
- 보안/컴플라이언스 요구사항 확인

### 단계 2: Design & Approval

- 아키텍처/권한 모델/네트워크 설계 문서화
- 용량 계획(CPU/RAM/DB/스토리지) 수립
- CAB(변경심의) 또는 내부 승인 완료

### 단계 3: Build & Install

- Staging 환경에 우선 설치
- 인증/연동/API 연결 테스트
- 설치 자동화 스크립트(Ansible, Helm values 등) 정리

### 단계 4: Validate & UAT

- 기능 테스트: 로그인, 권한, 워크플로 실행
- 비기능 테스트: 성능, 장애복구, 백업복원
- 사용자 수용 테스트(UAT) 및 개선사항 반영

### 단계 5: Production Cutover

- 변경 윈도우에 맞춘 운영 반영
- 모니터링/알림 임계치 활성화
- 초기 1~2주 집중 모니터링(하이퍼케어)

### 단계 6: Operate & Improve

- 월간 점검(패치, 계정, 접근 로그, 용량)
- 분기별 연계 시스템 품질 리뷰
- 운영 이슈 기반 워크플로 지속 개선

---

## 5) 설치 후 검증 체크리스트

- [ ] HTTPS로 UI 정상 접속
- [ ] 관리자 및 일반 사용자 권한 분리 동작 확인
- [ ] 주요 연동(API/웹훅/SSO) 성공 로그 확인
- [ ] 감사 로그 누락 없이 저장 확인
- [ ] 백업/복원 리허설 완료

---

## 6) 트러블슈팅 포인트

- **UI 접속 불가**: DNS, 인증서 체인, 인그레스/리버스프록시 설정 확인
- **SSO 로그인 실패**: IdP 메타데이터, 클레임 매핑, 시간 동기화 확인
- **연동 API 실패**: 토큰 만료, 스코프 권한, 방화벽 egress 정책 확인
- **성능 저하**: DB 연결풀, 스토리지 IOPS, 배치 작업 시간대 분산 검토

---

## 7) 운영 인수인계(핸드오버) 산출물

- 설치 아키텍처 다이어그램
- 환경 변수/시크릿 관리 정책서
- 백업/복원 Runbook
- 장애 대응 Runbook(알림 임계치 포함)
- 월간 운영 점검 체크리스트
